## Step 1: 给 M6 项目接入自动化工作流
在前面项目的基础上，你需要把 Issue → Agent → PR 的流程接起来，让至少 3 个 PR 可以通过这条流转。  
这一步的重点不是“自动化看起来很酷”，而是让重复性的协作动作变成标准流程。
场景对比：手工处理每个 Issue 都要重新解释上下文；自动化工作流则把需求、执行和审查固定下来。

## Step 2: 回顾你从 M1 到 M7 学到了什么
到了这个阶段，你应该能清楚说出自己如何从“会用 AI”走到“会设计协作系统”。  
回顾时不要只说感受，要具体说清楚：哪些原理最常用、哪些地方最容易踩坑、哪些习惯真正改变了你的项目推进方式。
场景对比：如果你只能说“我学会了很多工具”，说明还没真正把方法论串起来；如果能说清楚四大原则如何改变项目推进，那就已经进入系统化回顾。
