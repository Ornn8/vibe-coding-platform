## Step 1: 看懂什么叫一个完整的小项目
这个模块项目不追求复杂功能，而追求完整交付。一个完整的静态网站至少要有清晰主题、明确受众、两到三个页面或版块，以及一个可以向别人展示的入口。它可以是个人介绍站、课程专题页、读书会主页，也可以是一个小工具展示页。

评估项目好坏的标准不是“炫不炫”，而是你有没有把目标说清楚、有没有按计划推进、有没有保留 prompt 记录。我们从第一模块就开始强调这些，是为了让你后面做更复杂项目时，已经有闭环交付的基本习惯。

对比一下就能看出差别：

| 示例 | 是否完整 |
|---|---|
| 只有一个首页、没有说明用途 | 不完整 |
| 有首页、关于页、联系页，并能说明受众 | 更接近完整交付 |

## Step 2: 在开工前写项目计划
你的计划至少包含四项内容: 项目主题、目标受众、页面结构、视觉方向。页面结构不需要复杂，写成“首页 / 关于 / 联系方式”这种粒度即可。视觉方向也不需要会设计，描述成“极简、专业、暖色”就足够。

如果你跳过这一步，AI 很容易在第一轮就给你一个看似完整但其实方向模糊的页面，后面越改越散。把计划写出来的意义，是让你在每轮迭代时都有一个稳定参考。

可以把计划写成这个样子：

```text
主题：我的学习笔记站
受众：同学和朋友
页面：主页 / 笔记页 / 联系页
风格：简洁、安静、偏浅色
```

## Step 3: 执行、记录与提交
正式制作时，请把 prompt 记录和页面结果一起保留下来。最少记录三类信息: 第一轮 prompt、至少两轮迭代修改、最终展示链接或截图。如果你没法部署，也可以用截图和简短说明交付，但必须能让别人看懂你做了什么、怎么做的。

这一步的重点不是证明你已经很强，而是证明你已经开始用更稳定的节奏工作。后面所有模块项目都会在这个基础上继续加深。

如果你要检查自己有没有做全，可以用这个小清单:

| 交付物 | 是否保留 |
|---|---|
| 首轮 prompt | 是 / 否 |
| 两轮迭代记录 | 是 / 否 |
| 最终页面链接或截图 | 是 / 否 |

## 本单元小结

- 完整项目强调的是闭环，不是复杂度。
- 计划是执行基线，不是额外负担。
- 交付时要同时提交结果和过程记录。

## 方法论回顾

本单元第一次完整实践了“规划驱动”与“闭环交付”: 先计划、再执行、再提交可复盘材料。
