## Step 1: 先生成一个能展示的页面
这一步的目标不是做复杂产品，而是建立“描述目标 -> 看结果 -> 再修正”的身体记忆。你可以用个人介绍页、作品页或课程 landing page 作为题目，但在发 prompt 前先写三行: 页面给谁看、核心信息是什么、希望是什么风格。只要这三件事说清楚，AI 的首版质量通常就会高很多。

首轮 prompt 不要贪多。写清楚页面主题、三个内容区块和大致视觉方向就够了。先拿到一个可运行页面，比一开始追求完美更重要。

先看一个 before / after：

```text
差：帮我做一个好看的主页。
好：帮我做一个面向未来雇主的个人介绍页，首版先有自我介绍、项目展示和联系方式三个区块。
```

```text
请帮我生成一个个人介绍页。
目标用户：未来的合作伙伴和雇主。
核心信息：我是谁、我会什么、我最近做过什么。
风格：简洁、现代、移动端优先。
页面结构：Hero、技能卡片、项目展示、联系区。
```

## Step 2: 学会做第二轮和第三轮迭代
很多初学者的问题不是首版做不出来，而是第二轮开始失控。原因通常是反馈太模糊，例如“再高级一点”“更好看一些”。更有效的做法是针对具体问题发指令: 调整主色、增加 CTA 区、把卡片改成两列、提升移动端可读性。AI 更擅长处理明确的局部修改，而不是猜你心里的审美。

每一轮迭代都要回答三个问题: 我不满意哪里？我希望怎么改？这轮改完后我怎么判断是否更好？这就是最轻量的 Verify。

比如这两种说法的效果差很多:

```text
差：再高级一点。
好：把首页主色调改成更稳重的深蓝，增加一个明确的“联系我”按钮，并保证手机上第一屏不需要滚动太多。
```

## Step 3: 看懂 AI 产出了什么
你现在不需要完全理解所有文件，但需要学会观察结果。页面里通常会有入口文件、组件文件、样式文件和资源目录。哪怕只是粗略判断“这个文件负责结构、那个文件负责样式”，你已经开始建立对 AI 产出的控制感。

这个观察习惯很关键。后面进入 IDE 或 CLI 工具时，你要做的不只是让 AI 生成，还要判断它改了哪里、为什么这么改、下一步应不应该继续。今天的“看一眼文件结构”，其实是在为之后的工程化协作做准备。

| 文件/目录 | 可能负责的事 |
|---|---|
| `page` / `index` | 页面结构 |
| `components` | 可复用区块 |
| `styles` / `css` | 样式 |
| `public` | 图片与静态资源 |

## 本单元小结

- 先写三行小计划，再发第一轮 prompt。
- 第二轮开始要针对具体问题做明确修改。
- 观察产出结构，是从“会用”走向“能控制”的开始。

## 方法论回顾

本单元继续练习规划驱动: 先写目标，再迭代，再检查结果，而不是只追求一次生成成功。
