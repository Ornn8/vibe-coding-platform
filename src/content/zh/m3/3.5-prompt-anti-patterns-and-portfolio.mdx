## Step 1: 识别常见反模式
最常见的反模式包括三类: 模糊、贪心、无约束。模糊会让 AI 猜，贪心会让它摊薄重点，无约束会让它跑到你没预期的方向。再往上还有上下文污染和幻觉引导，也就是把错误信息当背景反复强化。

```text
反模式：帮我优化一下这个页面。
更好的写法：请只检查移动端导航、按钮间距和标题层级，不要改其他内容。
```

## Step 2: 学会做 Prompt 复盘
复盘时要看三件事: 原始目标是什么、prompt 哪部分导致偏差、下一次应该如何改。这个动作会让你逐渐摆脱“靠感觉写 prompt”，转向有证据、有版本感的改进方式。

一旦你开始这样复盘，你积累的就不再只是聊天记录，而是可重复使用的方法。

## Step 3: 建立个人 Prompt Portfolio
Portfolio 可以按任务类型分类，例如内容生成、代码修改、项目审查、需求拆解。每一条 prompt 除了正文，还要标注适用场景、常见坑和验证方式。这样它们才是资产，而不是历史记录。

到这里为止，你已经从“会写提示词”走到了“会管理 prompt 资产”。这会直接为 M4 的工作流阶段做准备。
