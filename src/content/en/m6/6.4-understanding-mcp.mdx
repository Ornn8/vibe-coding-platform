<Callout title="Method Focus" tone="success">
MCP is not just a list of tool names. It is the capability layer that upgrades AI from "only talking about the world" to "actually reaching real external systems."
</Callout>

## Step 1: Think of MCP as a new ability for AI
Compare two assistants:

| State | What AI can do |
|---|---|
| No MCP | It can only answer from the chat and the context already in memory |
| GitHub MCP connected | It can read the real repository, issues, and pull requests directly |

That is the most intuitive value of MCP. It does not merely make AI "better at talking." It gives AI a way to touch the repo, docs, database, or another real system.

```text
Before: AI has to guess what is in your repository.
After: AI can inspect the repository directly, then act from real evidence.
```

## Step 2: Connection alone does not make usage stable
Even when AI can connect to GitHub, a database, or a docs platform, it still might not know how to use that connection safely or consistently.

| Only MCP | MCP plus Skill |
|---|---|
| AI can reach the system | AI knows when to reach it, what to read first, and how to verify the result |
| Easy to over-query or misuse | Easier to stay inside the intended workflow |

The strongest pattern is not "MCP only." It is "MCP provides capability, Skill defines method."

<Callout title="Scenario: why access is still not enough" tone="warning">
Situation: AI can access the repository, but every time it inspects files in a different order and reports results inconsistently. Action: add a Skill that explains when to inspect, what to inspect first, and how to summarize findings. Why: capability without method is difficult to reuse safely.
</Callout>

## Step 3: The real value is the full workflow, not one call
A useful MCP workflow usually has four stages:

| Stage | Question to answer |
|---|---|
| Trigger | When should this MCP capability be used? |
| Access | What should AI read or write? |
| Process | How should the result be organized? |
| Verify | Who confirms the result is correct? |

If you only care about "Can it connect?", you will miss the more important question: "Once connected, how does this become a repeatable workflow?"

## Summary
- MCP gives AI a new way to access real external systems.
- Access still needs workflow guidance, which is where Skills fit.
- Stable glue coding comes from the full chain, not just a single successful connection.

## Methodology Review
Glue coding grows outward here: MCP carries the connection to the external world, while Skills keep that connection safe, useful, and repeatable.
