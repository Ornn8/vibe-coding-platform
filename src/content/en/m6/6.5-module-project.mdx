<Callout title="Project Focus" tone="warning">
The point of this project is not to make the app bigger. It is to make the capability system around the app more complete.
</Callout>

## Step 1: Plan the toolchain first

Before you start, write down what belongs in `CLAUDE.md`, which repeated tasks should become Skills, which outside systems need MCP, and where the memory-bank will live. Without that map, the system will be assembled in a messy, reactive way.

### Scenario

Before:
- Tools are added whenever a problem appears
- The project slowly accumulates disconnected habits

After:
- The toolchain is planned as a system
- Each asset has a clear job and a clear boundary

<Exercise id="ex-6.5.1" level="L1" title="Map the toolchain" description="Write down the project-level rules, the Skills you need, and the external connections your project requires." />

<Separator />

## Step 2: Turn configuration into real assets

This project should include at least one `CLAUDE.md`, two self-authored Skills, and one MCP integration or equivalent connection plan. Those are not decorative artifacts. They need to be used in real delivery work.

### Scenario

The curriculum's requirement is not "write the files and move on." It is "use them in actual delivery." That is why the result must show the assets working together rather than sitting in separate folders.

<Exercise id="ex-6.5.2" level="L3" title="Check asset completeness" description="Review the four asset types - CLAUDE.md, Skills, MCP, and memory-bank - and record what is already done versus missing." />

<Separator />

## Step 3: Deliver something real and evaluate the system

The final submission is not only the product result. It is also evidence that this toolchain helped you deliver more reliably. You should be able to explain which asset reduced the most friction and which part still feels too heavy.

### Scenario

A good retrospective compares two things:
- what the app gained
- what the workflow gained

If the toolchain improved delivery quality, that result should be visible in the project and in the review.

<Exercise id="ex-6.5.3" level="L3" title="Submit the toolchain project pack" description="Submit the code, CLAUDE.md, at least two self-authored Skills, at least one MCP integration, and the memory-bank, together with a short retrospective." />

## Summary
- Plan the capability system before configuring it.
- Configuration only matters if it changes real work.
- The retrospective should judge whether the toolchain actually reduced delivery cost.

## Methodology Review
This unit ties glue coding to context stabilization: tools, guidance, and memory only matter when they improve the next round of work.
