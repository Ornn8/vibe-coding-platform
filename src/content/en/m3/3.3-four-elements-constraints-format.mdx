<Callout type="methodology" title="Methodology Focus">
Constraints and output format turn a request into an instruction. Without them, AI often gives you something plausible but not actually usable.
</Callout>

## Step 1: Set the Boundaries

Constraints can be technical, stylistic, or behavioral. They tell AI what not to do, what to prioritize, and where the limits are. A prompt without constraints often produces output that is broad but not useful.

The point is not to make the prompt longer. The point is to make it harder for the model to drift.

Examples of useful constraints:

- Technical: "Use the existing MDX structure and do not change route names"
- Style: "Keep the tone calm and instructional"
- Behavioral: "Ask for clarification before editing if the requirement is ambiguous"
- Negative: "Do not introduce new libraries"

## Step 2: Specify the Output Format

Tell AI exactly what the result should look like. Do you want code, a checklist, a short note, a table, or a markdown brief? Format matters because it shapes how the information can be used next.

If you do not define the format, you may still get a good answer, but not necessarily the answer you can actually use.

| Request | Better format |
|--------|---------------|
| "Summarize the project" | A short markdown brief with headings |
| "Show me the next steps" | A numbered checklist |
| "Refactor this" | Code blocks plus a short change log |

## Step 3: Combine the Four Elements

When role, task, constraints, and format work together, the prompt becomes much more reliable. This is where you stop merely describing intent and start producing instructions that AI can follow with far less ambiguity.

That combination is the full four-element structure.

<Separator />

## Step 4: Check the Prompt Before You Send It

A useful prompt still benefits from a final review. Check whether the role is specific, the task is measurable, the constraints are real, and the format is explicit.

If one of the four parts is missing, the prompt is probably still too loose.
