## Step 1: Review the Four Core Principles

By this point you have practiced planning-driven learning, context stabilization, glue coding, and closed-loop delivery. They are not separate theories. They are different expressions of the same collaboration model.

The goal here is not memorization. It is to be able to connect each principle to something you actually did.

<Exercise id="ex-4.5.1.1" level="L2" title="Write a methodology reflection" description="For each of the four principles, write one concrete way it showed up in your project work." />

<Separator />

## Step 2: Know What the Module Project Must Include

The M4 project is not just code or a page. It also includes process artifacts: PRD, Implementation Plan, `CLAUDE.md`, and `progress.md`. That way reviewers can see that the result was produced through a repeatable process, not by luck.

This is the point where a learner starts acting more like an agentic builder.

<Exercise id="ex-4.5.2.1" level="L3" title="Check the delivery checklist" description="Review your project package and confirm that the code, PRD, plan, CLAUDE.md, and progress log are all present." />
