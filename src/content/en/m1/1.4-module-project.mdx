## Step 1: Choose a Small but Complete Project Topic
This unit does not reward complex functionality. It rewards complete delivery. A strong M1 project should fit inside one to two pages or three to four clear sections, while still making the topic, audience, and next action obvious.

Here are three model project directions:

| Topic | Best for | First-prompt direction | What the final result should feel like |
|---|---|---|---|
| Personal portfolio site | Interns, freelancers, job seekers | Highlight intro, projects, contact | A visitor quickly understands who you are |
| Reading-club event page | Student groups, clubs | Highlight event time, flow, signup | A visitor quickly decides whether to join |
| Small-shop menu page | Pop-up shops, cafes, food booths | Highlight categories, prices, featured items | A visitor quickly understands what to order |

<Callout type="scenario" title="How to notice when the topic is too big">
Situation: you want to build a full learning platform or a full blog system. Action: shrink the idea into a static showcase version and keep only the most important sections. Why: M1 is about complete delivery discipline, not feature depth.
</Callout>

## Step 2: Turn the Plan into an AI-Readable Build Brief
Before you start building, write a one-page project summary. At minimum, define the topic, audience, page structure, visual direction, and success standard.

```text
Project: reading club event page
Audience: students joining for the first time
Structure: Hero, highlights, event flow, signup section
Style: warm, clear, mobile-first
Success standard: a visitor can understand time, place, and signup path within 30 seconds
```

Then convert that into the first build prompt so AI is executing a defined plan instead of guessing your intent.

```text
Build a reading-club event page for first-time student visitors.
Include a Hero, highlights, event flow, and signup section.
Keep it warm, clear, and mobile-first.
A visitor should understand the time, place, and signup path within 30 seconds.
```

```text
Before: Build a homepage for my reading club.
After: Build a reading-club event page for first-time student visitors with Hero, highlights, event flow, and signup, so the time, place, and signup path are obvious in 30 seconds.
```

## Step 3: Generate, Iterate, and Assemble the Project Package
When you submit this project, the output is not just the final page. It is the result plus the process evidence. Keep at least three things: the first prompt, two rounds of iteration notes, and the final link or screenshots.

Use this delivery checklist:

| Artifact | Completion standard |
|---|---|
| First prompt | Audience, structure, and style are all visible |
| Two iteration notes | Each round changes one or two clear problems |
| Final result | You have a live link or a full screenshot set |
| Retrospective | You can explain what followed the plan and what changed |

<Callout type="scenario" title="Only the result vs result plus process">
Wrong move: submit one final screenshot and nothing else. Better move: submit the first prompt, iteration notes, and final result together. The difference is whether the work can be reviewed and learned from.
</Callout>

- Real delivery includes both the result and the process evidence.
