## MODIFIED Requirements

### Requirement: M4 planning-workflow lessons SHALL be fully authored in Chinese MDX
The system SHALL include complete Chinese MDX lesson content for the M4 learning arc as five v3-aligned Unit pages covering PRD plus implementation plans, the PEV loop and Ask/Plan modes, Claude Code onboarding, a full CRUD execution walkthrough, and a methodology-review project closeout.

#### Scenario: Learner opens an M4 lesson
- **WHEN** a learner visits any Chinese M4 lesson route
- **THEN** the corresponding MDX file SHALL contain complete unit-style instructional content instead of isolated topic stubs

#### Scenario: M4 lessons stay workflow-oriented
- **WHEN** an M4 lesson is reviewed
- **THEN** it SHALL present a repeatable workflow, artifact, or execution loop that the learner can apply during AI-assisted building

### Requirement: M4 lessons SHALL bridge from concept learning into execution discipline
Each Chinese M4 lesson SHALL make planning behavior operational by showing how to turn ideas into artifacts, how to verify AI outputs, and how to use terminal-based coding agents responsibly within the current repository-based workflow.

#### Scenario: Planning and execution units connect clearly
- **WHEN** the M4 Unit 4.1 through 4.4 lessons are reviewed together
- **THEN** they SHALL distinguish what to build, how to sequence the work, how to verify progress, and how to execute inside Claude Code

#### Scenario: Module closeout reinforces methodology
- **WHEN** the M4 Unit 4.5 lesson is reviewed
- **THEN** it SHALL ask the learner to submit both the project output and the planning artifacts used to produce it
